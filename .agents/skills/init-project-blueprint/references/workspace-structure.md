# Workspace Structure Reference

This file documents the canonical `.agent/` workspace layout representing the high-granularity standard.
The SKILL.md references this when generating `project_map.md` or onboarding content.

## Full Directory Tree

```
{project-root}/
├── AGENT.md                              # Brain Map — first file any agent reads
└── .agent/
    ├── onboarding.md                     # Comprehensive Orientation (Stack, Setup, Git Flow)
    ├── facts/                            # Static knowledge (agents: READ-ONLY)
    │   ├── glossary.md                   # Domain terms, acronyms, abbreviations
    │   ├── company_facts.md              # Company background, industry, compliance context
    │   └── project_map.md                # This directory tree + layer explanations
    ├── memory/                           # Dynamic knowledge (agents: READ & WRITE)
    │   ├── sessions/                     # Collaboration session logs
    │   │   └── YYYY-MM-DD_sync.md        # One file per session
    │   ├── user_preferences.md           # Timezone, language style, output preferences
    │   └── long_term_memory.md           # Key decisions extracted from past sessions
    ├── rules/                            # Hard constraints and operational standards
    │   ├── constraints.md                # Non-negotiable system rules (security, compliance)
    │   ├── design_system.md              # UI/UX standards, tone of voice, component rules
    │   └── developer_guide.md            # Code conventions, branching, testing, setup
    ├── workflows/                        # SOPs — step-by-step operating procedures
    │   └── module_creation_sop.md        # End-to-end SOP for creating a new module
    ├── scripts/                          # Utility scripts (Bash, CI/CD helpers)
    ├── reference/                        # Technical specifications (High-Granularity Standard)
    │   ├── 01_feature_list_mvp.md        # Granular, screen-level feature list grouped by module
    │   ├── 02_business_process_mvp.md    # Cross-module lifecycles (Mermaid diagrams)
    │   ├── 03_activity_diagram_mvp.md    # Deep logic & atomic transactions (Mermaid swimlanes)
    │   ├── 04_database_schema_mvp.md     # Data model, column types, schema notes (Mermaid ERD)
    │   └── 05_use_case_diagram.md        # Use case narratives & actor-system diagrams
    ├── skills/                           # Modular agent capabilities (micro-agents)
    │   └── {skill-name}/
    │       ├── SKILL.md                  # Capability description and trigger conditions
    │       ├── prompts/                  # Skill-specific system prompts
    │       ├── resources/                # Reference material for this skill
    │       └── tools/                    # Operational scripts for this skill
    ├── tasks/                            # Work management (agents: READ & WRITE)
    │   ├── backlog.md                    # Upcoming task queue
    │   └── in_progress.md                # Currently active tasks log
    └── meetings/                         # Discussion records (agents: READ & WRITE)
        ├── agendas.md                    # Topics for upcoming meetings
        └── logs/                         # Minutes of meeting (MoM)
```

## Layer Purpose Summary

| Layer        | Who Writes               | Who Reads          | Purpose                       |
| ------------ | ------------------------ | ------------------ | ----------------------------- |
| `facts/`     | Bootstrap skill (once)   | Agents (read-only) | Stable ground truth           |
| `memory/`    | Agents (ongoing)         | Agents             | Evolving project knowledge    |
| `rules/`     | Bootstrap skill + humans | Agents (read-only) | Non-negotiable standards      |
| `workflows/` | Bootstrap skill + humans | Agents             | Step-by-step SOPs             |
| `reference/` | Bootstrap skill + humans | Agents + devs      | Technical specifications      |
| `tasks/`     | Agents + humans          | Agents             | Work queue management         |
| `meetings/`  | Agents + humans          | Agents             | Decision & discussion log     |
| `skills/`    | Humans                   | Agents             | Modular capability extensions |
| `scripts/`   | Humans                   | Agents + CI        | Automation utilities          |

## Notes for Workspace Generation

- The `memory/`, `scripts/`, `skills/`, `meetings/`, and `reference/` directories are **not generated by the bootstrap skill** unless reference documents contain content that populates them. Create them as empty stubs with a `README.md` placeholder.
- The `reference/` diagrams (Mermaid) are generated strictly adhering to the Mermaid Standards.
- Never pre-populate `memory/` with invented session logs.
